# キャラクター・画像をオリジナルに変える（やることは2つだけ）

迷ったら **このファイルだけ** を見てください。

| やること | どこで |
|----------|--------|
| ① 画像ファイルを置く | フォルダ `public/images/characters/` |
| ② 名前・文章・画像のファイル名を書く | ファイル `src/data/characters.ts` |
| ③ 最初のTOPのスケジュール・メモ・見出し | ファイル `src/data/topLanding.ts` |

ブラウザに表示される「キャラ名・プロフィール・どの画像を使うか」は、すべて `characters.ts` に書いてあります。最初に開く `/` の右カラム文言は `topLanding.ts` です。

---

## ① 画像を置く場所（フォルダ）

エクスプローラーまたは Cursor の左のファイル一覧で、次のフォルダを開きます。

```text
homepageA
  └── public
        └── images
              └── characters   ← ここに JPG / PNG / WEBP などを入れる
```

**ルール:** ファイル名は英数字がおすすめです（日本語ファイル名は環境によってトラブルになりやすいです）。

**例（1人目のキャラ用）:**

| 用途 | おすすめのファイル名 | サイトでの使われ方 |
|------|----------------------|----------------------|
| トップの大きな背景 | `character01-bg.jpg` | トップページ全画面 |
| 詳細ページのメイン絵 | `character01-main.jpg` | 詳細ページ左の大きい画像 |
| 下部メニューの小さな画像（任意） | `character01-thumb.jpg` | 足元のキャラ切替サムネ。省略すると背景と同じ |
| ギャラリー1枚目 | `character01-gallery-01.jpg` | 詳細ページ下部のギャラリー |
| ギャラリー2枚目 | `character01-gallery-02.jpg` | 同上（好きな枚数だけ増やせる） |

2人目なら `character02-bg.jpg` のように `02` にそろえると管理しやすいです。

## ② 文章と画像パスを書くファイル

次のファイルを開きます。

```text
src/data/characters.ts
```

中に `characters: Character[] = [` とあり、そのあと `{` から `},` までが **1人分のキャラ** です。今は「01」「02」「03」の3人分あります。

### 各項目が画面のどこに出るか

| `characters.ts` の名前 | 画面での表示場所 |
|-------------------------|------------------|
| `id` | URL になります（`"01"` → 詳細は `/characters/01`）。**英数字だけ**がおすすめ |
| `name` | キャラクター名（大きく表示） |
| `nameReading` | 読み（ふりがななど） |
| `shortTagline` | トップのキャッチのような短い一行 |
| `heroDescription` | トップの短い説明文 |
| `age` | 年齢（数字だけ。例: `19`） |
| `height` | 身長（文字で。例: `"162cm"`）※ダブルクォート `"` で囲む |
| `likes` | 好きなもの |
| `catchphrase` | キャッチコピー |
| `introduction` | 詳細ページの「紹介文」。段落を変えたいときは `\n\n`（改行2つ）で区切る |
| `imageBg` | トップ背景用の画像 |
| `imageMain` | 詳細ページのメイン画像 |
| `imageThumb` | 下部サムネ用（無くてよい。その場合はこの行ごと削除して OK） |
| `gallery` | ギャラリー用画像のリスト（`[ ]` の中に `"パス"` をカンマ区切りで並べる） |

### 画像の書き方（ローカルファイルにしたとき）

`https://...` のような長い URL をやめて、**必ず先頭が `/` から始まるパス**にします。

置いたファイルが  
`public/images/characters/my-hero-bg.jpg`  
なら、データでは次のように書きます。

```ts
imageBg: "/images/characters/my-hero-bg.jpg",
```

`public` というフォルダ名は **URL に書かない** のがポイントです。`/images/...` だけで OK です。

### コピペ用の例（1人分のイメージ）

```ts
{
  id: "01",
  name: "あなたのキャラ名",
  nameReading: "よみがな",
  shortTagline: "トップに出す短いキャッチ",
  heroDescription: "トップに出す一言説明。",
  age: 20,
  height: "165cm",
  likes: "猫、コーヒー",
  catchphrase: "決め台詞のような一文",
  introduction: "長めの紹介。段落を分けたいときはこう書く。\n\n二段落目はここ。",
  imageBg: "/images/characters/character01-bg.jpg",
  imageMain: "/images/characters/character01-main.jpg",
  imageThumb: "/images/characters/character01-thumb.jpg",
  gallery: [
    "/images/characters/character01-gallery-01.jpg",
    "/images/characters/character01-gallery-02.jpg",
  ],
},
```

- ギャラリーを1枚だけにしたい → `gallery: ["/images/characters/character01-gallery-01.jpg"],`
- サムネを省略したい → `imageThumb` の行を削除（そのときは `imageBg` がサムネ代わりになります）

## 画像の解像度・容量の目安（大きすぎると出ないことがある）

`next/image` は表示前にサーバー側でリサイズします。**ファイル容量が大きい・画素数が極端に多い**と、処理が間に合わず真っ黒や壊れた表示になることがあります（タイムアウトやメモリの都合）。

### おすすめのサイズ（目安）

| 用途 | 長辺の目安 | ファイル容量の目安 | 形式 |
|------|------------|-------------------|------|
| トップ背景 `imageBg` | **1920〜2560px**（4K でも可だが要圧縮） | **〜2MB** 程度までが無難 | JPEG 品質 80〜90%、または **WebP** |
| 詳細メイン `imageMain` | **1200〜2000px** | **〜1.5MB** | 同上 |
| サムネ `imageThumb` | **400〜600px** | **〜200KB** | 同上 |
| ギャラリー | **1000〜1600px** | 各 **〜800KB** | 同上 |

印刷用の **数千ピクセル角・数十 MB の PSD 書き出しそのまま** は避け、Web 用に一度エクスポートし直すのが確実です。

### それでも出ないときに確認すること

1. **ブラウザの開発者ツール**（F12）→「ネットワーク」で `my-hero-bg` や `_next/image` が **赤い 400/500** になっていないか見る。
2. **パス**が `public/images/characters/my-hero-bg.jpg` に本当に存在するか、**拡張子の大文字小文字**が一致しているか（Windows はファイルシステム上は区別しないこともあるが、ビルド先では区別される場合あり）。
3. **CMYK の JPEG** は環境によって失敗することがある。**RGB で書き出し**直す。
4. このプロジェクトの `next.config.ts` では、画像最適化の **タイムアウトを 30 秒** に延ばしてあります。それでもダメなら **画素数・容量を下げる**のが正解です。

## よくあるつまずき

### Q. 保存したのに画像が出ない

- パスが `/images/characters/ファイル名` になっているか確認（`public` を付けない）。
- ファイル名のスペル・拡張子（`.jpg` / `.png`）が実物と一致しているか確認。

### Q. まだ Unsplash の写真が出る

`characters.ts` の `https://images.unsplash.com/...` を、すべて自分の `/images/characters/...` に書き換えてください。1箇所でも URL が残っていると、その画像が表示されます。

### Q. ローカル画像だけにしたい（Unsplash を完全にやめたい）

1. `characters.ts` の画像をすべて `/images/characters/...` にする。  
2. `next.config.ts` を開き、`images: { remotePatterns: [...] }` ごと削除して保存しても大丈夫です（ローカルだけなら不要）。

### Q. キャラを2人だけにしたい

`characters.ts` の配列から、いらない `{ ... },` のブロックを削除します。カンマの位置に注意（最後の1人のあとに余計な `,` がないか）。

### Q. キャラをもう1人足したい

配列の末尾に `,` を付けてから、もう一段 `{ ... },` を追加。`id` は他と被らないように（例: `"04"`）。

## 編集したあと

ターミナルで `npm run dev` を動かしているなら、ファイルを保存するとほぼそのまま反映されます。反映されないときは一度ターミナルを止めて（Ctrl+C）、もう一度 `npm run dev` を実行してください。

---

## まとめ（最短手順）

1. 画像を `public/images/characters/` にコピーする。  
2. `src/data/characters.ts` を開く。  
3. 名前・年齢・紹介文などの **文字** を自分用に変える。  
4. `imageBg` / `imageMain` / `gallery` などの **https://...** を **`/images/characters/自分のファイル名`** に変える。  
5. 保存してブラウザを更新する。

以上です。
