# キャラクター・スケジュールの編集ガイド（迷ったらこれだけ）

普段よく触る「キャラクターの追加・修正」と「スケジュールの編集」を、最短で分かるようにまとめています。

| やりたいこと | 開くファイル |
|----------|--------|
| ① 画像を置く | フォルダ `public/images/characters/` |
| ② キャラの追加・名前・文章・画像を変える | `src/data/characters.ts` |
| ③ トップのスケジュール（出勤表）を変える | `src/data/topLanding.ts` |
| ④ トップの見出し・説明文・背景を変える | `src/data/topLanding.ts` |

> 打ち間違い（存在しないキャラのIDや名前など）があると、`npm run dev` を動かしているターミナルに **警告メッセージ** が出ます。表示がおかしいときはまずターミナルを見てください。

---

## ① 画像を置く場所

```text
homepageA
  └── public
        └── images
              └── characters   ← ここに JPG / PNG / WEBP を入れる
```

**ルール:** ファイル名は英数字がおすすめ（日本語ファイル名は環境によってトラブルになりやすい）。

データ側では、置いたファイルを **先頭が `/` のパス** で書きます。`public` は書きません。

```text
置いた場所: public/images/characters/char08.png
データの書き方: "/images/characters/char08.png"
```

## ② キャラクターの追加・修正（`src/data/characters.ts`）

このファイルの `characters` 配列に、`defineCharacter({ ... })` が **1人分** です。

### キャラを1人足す（最短）

配列の中（テンプレートのコメント付近）に、次を足すだけ。**必須は `id` / `name` / `image` の3つ**だけです。

```ts
defineCharacter({
  id: "08",                                 // 必須: URL用の番号（他と重複NG）。例: /characters/08
  name: "新しいキャスト",                     // 必須: 表示名
  image: "/images/characters/char08.png",   // 必須: 代表画像（背景・立ち絵・サムネに共通で使われる）
}),
```

`image` を1枚指定すれば、背景・立ち絵・サムネすべてに使い回されます。個別に変えたいときだけ下記の画像項目を足します。

### 書ける項目の一覧（すべて任意。省略すると自動で非表示／空欄）

| 項目名 | 画面での表示場所 |
|-------------------------|------------------|
| `id` | URL になる（`"08"` → `/characters/08`）。**英数字だけ**がおすすめ・**重複NG** |
| `name` | キャラクター名（大きく表示） |
| `shortTagline` | トップ等で出る短い一行のキャッチ |
| `heroDescription` | トップで出る短い説明文 |
| `vrchatAge` | 詳細ページの「VRChat Age（在籍年数など）」 |
| `skills` | 詳細ページの「Skills（特技）」 |
| `likesDislikes` | 詳細ページの「Likes / Dislikes（好き / 苦手）」 |
| `contactSns` | 詳細ページの「Contact / SNS」 |
| `message` | 詳細ページの自由記述。段落を分けるときは `\n\n`（改行2つ）で区切る |
| `image` | 代表画像。下の各画像を省略したときのフォールバックになる |
| `imageBg` | トップ全画面背景（省略時は `image`） |
| `imageMain` | 詳細ページのメイン画像（省略時は `image`） |
| `imageThumb` | 下部サムネ（省略時は `imageBg`／`image`） |
| `gallery` | ギャラリー画像のリスト（`[ ]` の中に `"パス"` をカンマ区切り） |

> 詳細ページの `vrchatAge` / `skills` / `likesDislikes` / `contactSns` / `message` は、**空欄にした行は自動で表示されません**。書いた項目だけが出ます。

### しっかり書いた例（1人分）

```ts
defineCharacter({
  id: "08",
  name: "美月",
  shortTagline: "ひとことキャッチ。",
  heroDescription: "トップに出る短い説明。",
  vrchatAge: "在籍 1年目",
  skills: "聞き上手 / カクテル作り",
  likesDislikes: "好き: ○○ / 苦手: △△",
  contactSns: "X: @your_id",
  message: "自由記述のメッセージ。\n\n段落は空行（\\n\\n）で区切れます。",
  image: "/images/characters/char08.png",
  // 画像を個別に分けたいときだけ↓を足す（不要なら丸ごと省略でOK）
  // imageBg: "/images/characters/char08-bg.png",
  // imageMain: "/images/characters/char08-main.png",
  // imageThumb: "/images/characters/char08-thumb.png",
  gallery: [
    "/images/characters/char08.png",
    "/images/characters/char08.png",
  ],
}),
```

`characters.ts` の末尾には、コメントアウト済みの **コピペ用テンプレート** も用意してあります。

### キャラを減らす / 並べ替える

- 減らす: いらない `defineCharacter({ ... }),` のブロックを丸ごと削除（カンマの位置に注意）。
- 並べ替え: ブロックを上下に入れ替えるだけ。下部のキャラ選択の並び順に反映されます。

## ③ スケジュール（出勤表）の編集（`src/data/topLanding.ts`）

`topSchedule` 配列の `{ ... }` が **1予定** です。上から順に表示されます。

### キャストの指定は2通り（どちらでもOK）

**(A) `id` で指定（おすすめ）** — `characters.ts` の `id` を書きます。

```ts
{ date: "6/1", time: "20:00～26:00", id: "01", status: "出勤" },
```

- 名前は自動で表示され、キャラを改名しても自動で追従します。
- トップのスケジュール項目を **クリックするとそのキャストのページへ遷移** します。

**(B) `name` で指定** — 名前を直接書きます。

```ts
{ date: "6/1", time: "20:00～26:00", name: "美咲", status: "出勤" },
```

- `characters.ts` の `name` と一致すれば自動でリンクされます。
- 一致しない名前だと、ターミナルに警告が出ます（打ち間違い検知）。

### 各項目の意味

| 項目 | 必須? | 例 | 説明 |
|------|------|----|------|
| `date` | 必須 | `"6/1"` | 日付 |
| `time` | 任意 | `"20:00～26:00"` | 時間帯。省略すると非表示 |
| `id` | （id か name どちらか） | `"01"` | キャラID。推奨。名前は自動表示＆クリックで遷移 |
| `name` | （id か name どちらか） | `"美咲"` | キャスト名。id を書かないときは必須 |
| `status` | 任意 | `"出勤"` / `"店長"` / `"GUEST"` | 状態バッジ。省略すると非表示 |

- 予定を **増やす**: `{ ... },` の行をコピーして書き換える（何件でもOK）。
- 予定を **減らす**: 不要な `{ ... },` の行を消す。
- 件数が多くて入りきらない場合は、SCHEDULE 内で自動的にスクロールします。

## ④ トップの見出し・説明文・背景（`src/data/topLanding.ts`）

- `topLandingCopy.kicker` … 小さな飾り文字（上部）
- `topLandingCopy.title` … 大きなタイトル。`\n` を入れた箇所で改行
- `topLandingCopy.lead` … 説明文。`\n` を入れた箇所で改行（1〜3行程度）
- `topBackground` … トップ背景。**単色** か **画像** を選べます（ファイル内のコメント参照）。

```ts
// 単色にする
export const topBackground: TopBackground = { type: "color", color: "#0a0a0a" };

// 画像にする（public/images/ に置いたファイルを指定）
export const topBackground: TopBackground = {
  type: "image",
  src: "/images/top-bg.png",
  overlay: 0.5,        // 文字を読みやすくする暗さ（0=なし〜1=真っ黒）
  position: "center",
};
```

## 画像の解像度・容量の目安（大きすぎると表示されないことがある）

`next/image` は表示前にサーバー側でリサイズします。**容量が大きい・画素数が極端に多い**と、処理が間に合わず真っ黒や壊れた表示になることがあります。

| 用途 | 長辺の目安 | 容量の目安 | 形式 |
|------|------------|-------------------|------|
| トップ背景 `imageBg` | 1920〜2560px | 〜2MB | JPEG 品質80〜90% または WebP |
| 詳細メイン `imageMain` | 1200〜2000px | 〜1.5MB | 同上 |
| サムネ `imageThumb` | 400〜600px | 〜200KB | 同上 |
| ギャラリー | 1000〜1600px | 各〜800KB | 同上 |

印刷用の数千ピクセル角・数十MBのデータそのままは避け、Web用にエクスポートし直すのが確実です。

## よくあるつまずき

### Q. 保存したのに画像が出ない
- パスが `/images/characters/ファイル名` になっているか（`public` を付けない）。
- ファイル名のスペル・拡張子（`.jpg` / `.png`）が実物と一致しているか。

### Q. スケジュールの名前が「(名前未設定)」になる / クリックできない
- `id` で指定している場合: その `id` が `characters.ts` に存在するか確認。
- `name` で指定している場合: `characters.ts` の `name` と1文字も違わず一致しているか確認。
- ターミナル（`npm run dev` 実行中）に出る警告メッセージも確認してください。

### Q. キャラをもう1人足したい
- `characters.ts` の配列末尾に `defineCharacter({ ... }),` を追加。`id` は他と被らないように。

## 編集したあと

`npm run dev` を動かしていれば、ファイルを保存するとほぼそのまま反映されます。反映されないときは一度ターミナルを止めて（Ctrl+C）、もう一度 `npm run dev` を実行してください。

## まとめ（最短手順）

1. 画像を `public/images/characters/` にコピー。
2. キャラを変える → `src/data/characters.ts` の `defineCharacter({ ... })` を編集・追加。
3. 出勤表を変える → `src/data/topLanding.ts` の `topSchedule` を編集。
4. 保存してブラウザを更新。おかしいときはターミナルの警告を確認。
