# サイトの「文字（文言）」修正ガイド

サイトに出てくる文字を変えたいときの **場所の早見表** と **具体的な直し方** をまとめています。
（キャラクター・スケジュール・画像の追加は別ファイル [`キャラクター編集ガイド.md`](./キャラクター編集ガイド.md) を参照）

> 編集後、`npm run dev` を動かしていれば保存でほぼ即反映されます。反映されないときはターミナルを `Ctrl+C` で止めて、もう一度 `npm run dev`。

## 1. URLプレビュー（OGP / SNSサムネイル）

URLをSNSやチャットに貼ったときに出る「タイトル・説明・画像」です。

### 1-1. 文字（タイトル・説明） — `src/app/layout.tsx`

ファイル上部の次の2つを書き換えるのが基本です。これだけでタイトル・OGP・Twitterすべてに反映されます。

```ts
const SITE_NAME = "ORIGINAL CHARACTER ARCHIVE";          // サイトの題名（大見出し）
const SITE_DESCRIPTION = "オリジナルキャラクター…公式アーカイブ。"; // 説明文
```

さらに細かく直す場合、`metadata` の中：

| 項目 | 役割 |
|---|---|
| `title.default` | トップページの題名（初期値は `SITE_NAME`） |
| `title.template` | 各ページの題名の形。`"%s — CHARACTER ARCHIVE"` の `%s` にページ名が入る。**末尾の「CHARACTER ARCHIVE」を店名等に変更** |
| `applicationName` / `openGraph.siteName` | サイト名（SNSに表示される運営名） |
| `openGraph.title` / `twitter.title` | SNS用の題名 |
| `openGraph.description` / `twitter.description` | SNS用の説明 |

> **重要：本番URLの設定**
> `metadataBase: new URL("https://example.com")` が仮の値になっています。OGP画像は絶対URLで参照されるため、**Vercelの本番ドメインに変更**してください。
> 例：`metadataBase: new URL("https://your-project.vercel.app"),`

### 1-2. プレビュー画像（カードの絵柄） — `src/app/opengraph-image.tsx`

カードの画像（1200×630）はコードで自動生成しています。中の**文字**を直す場合：

| カード内の文字 | 該当箇所 |
|---|---|
| 上の小見出し `ORIGINAL CHARACTER` | `ORIGINAL CHARACTER` の行 |
| 中央の大きい `CHARACTER` / `ARCHIVE` | `<div>CHARACTER</div>` `<div>ARCHIVE</div>` |
| 下の英文 `A QUIET ARCHIVE OF ORIGINAL CHARACTERS` | 同名の行 |
| 読み上げ用の代替文 | ファイル4行目 `export const alt = "..."` |

- 背景色は `background:` の行、文字色は各 `color:` を編集。

**自作の画像（写真など）をそのまま使いたい場合**は、`opengraph-image.tsx` を削除し、画像を `public/`（例：`public/ogp.png`）に置いて `layout.tsx` の `openGraph` に追記：

```ts
openGraph: {
  // ...既存の設定...
  images: [{ url: "/ogp.png", width: 1200, height: 630 }],
},
```

### 1-3. 反映の注意（キャッシュ）

SNS各社は一度読んだプレビューをキャッシュします。直しても古いままのときは、各社のデバッガーで再取得してください。
- X：Card Validator
- Facebook / Threads：Sharing Debugger（「Scrape Again」）
- LINE / Discord：時間を置くと更新

## 2. ブラウザのタブに出るページ名

各ページのファイル冒頭にあります。例：

```ts
// src/app/about/page.tsx
export const metadata: Metadata = {
  title: "ABOUT",
};
```

`title` の値を変えると、タブ名が「（その文字）— CHARACTER ARCHIVE」になります（末尾は 1-1 の `template` で変更）。
対象：`about` / `contact` / `gallery` / `characters` / `spotlight` の各 `page.tsx`。

---

## 3. ヘッダー（左上ロゴ・メニュー） — `src/components/HoverTopNav.tsx`

### メニュー項目
ファイル上部の配列を編集します。`label` が画面の表示文字、`href` がリンク先です。

```ts
const links = [
  { href: "/characters", label: "CAST" },
  { href: "/gallery", label: "GALLERY" },
  { href: "/about", label: "ABOUT" },
  { href: "/contact", label: "CONTACT" },
];
```

- 文字を変える → `label` を変更（リンク先 `href` はそのままでOK）
- 項目を増減 → 行を追加・削除

### 左上のロゴ文字
`CHARACTER ARCHIVE` を `TOP` に変更済み。さらに変えたい場合は、`<Link href="/">` 内の `TOP` という文字を書き換えます。

## 5. フッター — `src/components/SiteFooter.tsx`

| 文字 | 該当箇所 |
|---|---|
| ロゴ文字 `CHARACTER ARCHIVE` | `<span>CHARACTER ARCHIVE</span>` |
| ロゴ下の説明文 | 「オリジナルキャラクターの世界観と…」の `<p>` |
| メニューリンク | `navLinks` 配列の `label` |
| SNSリンク | `socialLinks` 配列（`label` が表示名、`href` がURL） |
| コピーライト | `© {year} CHARACTER ARCHIVE` の `CHARACTER ARCHIVE` 部分（年は自動） |
| 右下の注記 | `ALL VISUALS ARE FOR DEMONSTRATION` |

> フッターは TOP（`/`）とスポットライト（`/spotlight`）では非表示です（`HIDE_ON` で制御）。

## 8. 各ページの本文

### ABOUT — `src/app/about/page.tsx`
- 小見出し `CONCEPT`、見出し `ABOUT`、本文の各 `<p>` を書き換え。

### CONTACT — `src/app/contact/page.tsx`
- 小見出し `INQUIRY`、見出し `CONTACT`、説明文。
- **メールアドレス**：`mailto:contact@example.com` と表示文字 `contact@example.com` の2か所を実際のアドレスに。
- **SNS**：`<a href="...">X (Twitter)</a>` などのURL・表示名を編集／追加・削除。

### CHARACTERS（一覧 ＝ ヘッダーの「CAST」を押した先のページ） — `src/components/CharactersIndexContent.tsx`
- 小見出し `INDEX`、見出し `CHARACTERS` を書き換え。
- **説明文**：次の一文がページ上部の説明です。`<p>` の中の文を書き換えてください。

```tsx
<p className="mt-5 max-w-lg text-sm font-extralight leading-relaxed text-white/55">
  世界観を支えるキャラクターの一覧。各カードから詳細プロフィールとギャラリーをご覧いただけます。
</p>
```

- カード内の `NO.◯◯`・`VIEW PROFILE` などの定型文もこのファイル内にあります。

### GALLERY — `src/app/gallery/page.tsx`
- 小見出し `VISUAL`、見出し `GALLERY`、説明文を書き換え。

### 404ページ — `src/app/not-found.tsx`
- `Page not found` や説明文、戻るボタンの `TOP` を書き換え。

## 困ったときのチェック

1. 保存したのに変わらない → ターミナル（`npm run dev`）を再起動。
2. 記号や引用符を消してしまうとエラーになることがある → `"文字"` の `"`（ダブルクォート）は消さない。行末の `,`（カンマ）も残す。
3. SNSのサムネだけ古い → 上記 1-3 のデバッガーで再取得。
